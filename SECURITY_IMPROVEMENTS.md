# Улучшения безопасности

## Выполненные изменения

### 1. Защита .env файлов (Высокий приоритет) ✅
- Обновлены `.gitignore` в корне и `/www`:
  - Добавлены паттерны: `.env`, `.env.local`, `.env.*.local`, `*.env`
  - Добавлена защита SQL-дампов: `*.sql`, `*.sql.dump`, `*.sql.gz`, `*.sql.bz2`
  - Добавлены директории бэкапов: `backups/`, `db_dumps/`, `database_backups/`
  - Добавлены файлы БД: `*.sqlite`, `*.sqlite3`, `*.db`

### 2. Rate Limiting для API (Высокий приоритет) ✅
- Реализована улучшенная функция `checkRateLimit()` в `auth_middleware.php`:
  - Защита от race conditions с помощью `flock()`
  - Разные лимиты для разных эндпоинтов:
    - login: 10 запросов/мин
    - register: 5 запросов/мин
    - profile: 20 запросов/мин
    - api endpoints: 30-60 запросов/мин
  - Логирование превышений лимита как security events
  - HTTP 429 ответ при превышении лимита

### 3. Централизованное логирование (Высокий приоритет) ✅
- Создан новый файл `api/logger.php`:
  - Класс `Logger` с уровнями: DEBUG, INFO, WARNING, ERROR, CRITICAL, SECURITY
  - Автоматическое создание директории `/logs` с защитой `.htaccess`
  - Раздельные логи для обычных событий и security событий
  - Маскирование чувствительных данных (password, token, secret, key)
  - Перехват фатальных ошибок и необработанных исключений
  - Аудит действия пользователей через `Logger::audit()`

### 4. Улучшение аутентификации ✅
- Обновлён `auth_middleware.php`:
  - Логирование всех попыток доступа без токена
  - Логирование невалидных токенов с IP и User-Agent
  - Интеграция с новой системой логирования

### 5. Улучшение login.php ✅
- Детальное логирование событий безопасности:
  - Попытки входа с пустыми учётными данными
  - Попытки входа для несуществующих пользователей
  - Неверные пароли с указанием user_id
  - Попытки входа неактивных пользователей
  - Успешные входы с аудитом
- Удалено дублирование функции `generate_secure_token()`

### 6. Защита директории логов ✅
- Создана директория `/workspace/www/logs/`
- Добавлен `.htaccess` с запретом доступа ко всем файлам

## Проверка работы

1. Проверить что `.env` не попадает в git:
   ```bash
   git check-ignore .env
   ```

2. Проверить работу rate limiting:
   - Сделать более 10 запросов к `/api/login.php` в минуту
   - Должен вернуться HTTP 429

3. Проверить логирование:
   - После попытки входа проверить файлы в `/workspace/www/logs/`
   - Security события должны быть в `security_YYYY-MM-DD.log`
